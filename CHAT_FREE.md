# Бесплатный чат (без платного тарифа)

Чат в приложении работает на **бесплатном тарифе Firebase (Spark)**. Деньги за чат платить не нужно.

---

## Пошаговая настройка чата

Сделайте шаги по порядку. После шага 4 сообщения должны доходить до пользователей (пока приложение открыто).

### Шаг 1. Открыть Firebase и Realtime Database

1. Откройте браузер и перейдите на [Firebase Console](https://console.firebase.google.com).
2. Выберите ваш проект (например **startdrive-573fa**).
3. В левом меню: **Build** → **Realtime Database**.

### Шаг 2. Создать базу (если её ещё нет)

1. Если видите кнопку **Create Database**:
   - Нажмите её.
   - Выберите регион (рекомендуется **europe-west1** или тот, что ближе к вам).
   - Режим правил: **Start in locked mode**.
   - Подтвердите создание.
2. Если база уже есть — переходите к шагу 3.

### Шаг 3. Указать URL базы в приложении

1. В Firebase Console на странице Realtime Database **скопируйте URL** — он показан вверху (например: `https://startdrive-573fa-default-rtdb.europe-west1.firebasedatabase.app`).
2. В проекте откройте файл:  
   **`app/src/main/java/com/example/startdrive/data/FirebasePaths.kt`**
3. Найдите константу **`REALTIME_DATABASE_URL`**.
4. Подставьте ваш URL в кавычки:
   - Если база в **europe-west1** (или другом регионе не США):  
     `const val REALTIME_DATABASE_URL: String? = "https://ВАШ-ПРОЕКТ-default-rtdb.europe-west1.firebasedatabase.app"`
   - Если база в **США (us-central1)** — задайте `null`:  
     `const val REALTIME_DATABASE_URL: String? = null`
5. Сохраните файл.

### Шаг 4. Опубликовать правила доступа

1. В Firebase Console в Realtime Database откройте вкладку **Rules**.
2. Удалите весь текст в редакторе правил.
3. Откройте в проекте файл **`database.rules.json`** (в корне) или **`database.rules.console.txt`** и скопируйте оттуда весь JSON (блок с `"rules": { "chats": ... }`).
4. Вставьте скопированное в редактор правил в консоли.
5. Нажмите **Publish**.
6. Дождитесь сообщения об успешном обновлении правил.

Без этих правил сообщения не сохраняются и не отображаются у получателя.

### Шаг 5. Собрать и установить приложение

1. В Android Studio откройте проект StartDrive.
2. Меню **Build** → **Run** (или зелёная кнопка Run).
3. Установите приложение на устройство или эмулятор.

Дополнительные серверы или деплой из консоли для чата не нужны.

### Шаг 6. Проверка

1. Войдите в приложение под **двумя разными пользователями** (например админ и инструктор или инструктор и курсант).
2. У первого: откройте кабинет → вкладка **Чат** → выберите второго пользователя из списка → отправьте сообщение.
3. У второго: откройте кабинет → вкладка **Чат** → выберите первого пользователя. Должно появиться отправленное сообщение.
4. Если при отправке появляется ошибка (Snackbar у админа или Toast у инструктора/курсанта) — вернитесь к шагам 3 и 4 (URL базы и правила).

---

## Что умеет бесплатный чат

- Отправка и получение сообщений в реальном времени.
- Сохранение переписки (при смене вкладки или контакта сообщения не пропадают).
- Звук при новом сообщении от собеседника (когда приложение открыто).
- Работает для админа, инструктора и курсанта.

---

## Ограничение бесплатного варианта

**Push-уведомления** (в шторку или на экран блокировки, когда приложение закрыто) в этом варианте **не приходят** — для них нужен платный тариф Blaze и развёртывание Cloud Functions. Пока пользователь в приложении и открыт чат — сообщения и звук работают без ограничений.

## Если сообщения не доходят

1. **Опубликованы ли правила?**  
   В Firebase Console → Realtime Database → **Rules** должно быть вставлено содержимое из **`database.rules.json`** и нажата кнопка **Publish**. Без этого запись и чтение чата блокируются.

2. **Совпадает ли URL базы с приложением?**  
   В Firebase Console откройте Realtime Database — вверху указан **URL** (например `https://ВАШ-ПРОЕКТ-default-rtdb.europe-west1.firebasedatabase.app`).  
   В проекте откройте **`app/src/main/java/.../data/FirebasePaths.kt`**: константа **`REALTIME_DATABASE_URL`** должна совпадать с этим URL. Если база создана в **США (us-central1)**, задайте **`REALTIME_DATABASE_URL = null`** — тогда приложение использует стандартный URL.

3. **Один и тот же проект**  
   В приложении должен использоваться тот же проект Firebase (тот же `google-services.json`), в котором создана Realtime Database и опубликованы правила.

4. **Оба пользователя залогинены**  
   Правила разрешают доступ только при `auth != null`. Оба отправитель и получатель должны быть авторизованы в приложении.

5. **Приложение открыто у получателя**  
   В бесплатном варианте сообщения приходят в реальном времени, пока приложение открыто (на любом экране). После входа в кабинет приложение подписывается на все чаты — новые сообщения должны появляться сразу после открытия чата с отправителем. Если отправитель видит ошибку при отправке (например, в чате админа показывается Snackbar) — проверьте правила и URL базы.
