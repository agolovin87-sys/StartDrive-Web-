# Подробная инструкция: развёртывание Cloud Functions для push-уведомлений чата

В этом документе по шагам описано, как развернуть функцию, которая отправляет push-уведомление (со звуком) получателю при новом сообщении в чате.

---

## Вариант без платного тарифа (Spark)

**Cloud Functions в Firebase доступны только на тарифе Blaze.** На бесплатном тарифе Spark их подключить нельзя.

### Что работает бесплатно (без Blaze)

- **Сообщения доходят в реальном времени**, пока приложение открыто:
  - Realtime Database и слушатели работают на Spark.
  - При открытии вкладки «Чат» приложение подписывается на все ваши диалоги.
  - Новые сообщения сразу отображаются в чате и можно включить **звук при новом сообщении** (уже реализовано в приложении).

- **Правила Realtime Database** можно настроить и публиковать на Spark — чат будет сохраняться и синхронизироваться между пользователями.

### Чего не будет без Blaze

- **Push-уведомления, когда приложение в фоне или закрыто** — для этого нужен сервер, который отправит FCM при новом сообщении. В нашем проекте эту роль выполняет Cloud Function, а она работает только на Blaze. Без неё уведомления на заблокированный экран или в шторку не придут.

### Итог

- Если нужны только чат и звук при новом сообщении **в открытом приложении** — платный тариф не нужен, всё уже работает на Spark.
- Если нужны push, когда приложение закрыто или свернуто — нужен тариф Blaze и развёртывание Cloud Functions по инструкции ниже (или свой сервер вне Firebase).

---

## Что такое Cloud Functions и зачем они нужны

- **Cloud Functions** — это код, который выполняется на серверах Google в ответ на события (например, новое сообщение в базе).
- В нашем случае: когда в Realtime Database появляется новое сообщение по пути `chats/{roomId}/messages/{messageId}`, функция:
  1. Определяет получателя по `roomId` и отправителя по полю `senderId` в сообщении.
  2. Читает из Firestore FCM-токен получателя (поле `fcmToken` в документе пользователя).
  3. Отправляет на этот токен push-уведомление с заголовком «Новое сообщение» и текстом сообщения (со звуком).

Без этой функции приложение само по себе не может отправить push другому пользователю при новом сообщении — для этого нужен сервер (здесь его роль выполняет Cloud Function).

## 1. Требования перед началом

### 1.1. Node.js

Firebase CLI и код функций работают на Node.js. Нужна версия **18** или **20** (рекомендуется 20).

**Проверка в терминале:**

```bash
node --version
```

Должно быть что-то вроде `v20.x.x` или `v18.x.x`. Если команды нет или версия старая:

1. Зайдите на [https://nodejs.org](https://nodejs.org).
2. Скачайте и установите LTS-версию (например 20.x).
3. Перезапустите терминал и снова выполните `node --version`.

### 1.2. Тариф Blaze в Firebase

Cloud Functions доступны только на платном тарифе **Blaze** (оплата по факту использования).

**Как перейти на Blaze:**

1. Откройте [Firebase Console](https://console.firebase.google.com).
2. Выберите проект приложения (например **startdrive-573fa**).
3. Слева внизу нажмите на иконку **шестерёнки** → **Использование и выставление счетов** (или **Usage and billing**).
4. Нажмите **Перейти на тариф Blaze** (или **Modify plan** → **Blaze**).
5. Привяжите платёжный аккаунт Google (карта). Для малого числа вызовов функций списания обычно минимальны (есть бесплатный ежемесячный лимит).

### 1.3. Realtime Database создана

В консоли Firebase: **Build** → **Realtime Database**. Если база ещё не создана — нажмите **Create Database**, выберите регион (например **europe-west1**) и режим правил (потом можно подставить правила из `database.rules.json`).

Запомните регион базы: он понадобится для настройки функции (раздел 5).

## 2. Установка Firebase CLI

Firebase CLI — программа для входа в аккаунт, выбора проекта и развёртывания функций из командной строки.

**В терминале (PowerShell или cmd) выполните:**

```bash
npm install -g firebase-tools
```

- `npm` — менеджер пакетов Node.js (ставится вместе с Node.js).
- `-g` — установка глобально, чтобы команда `firebase` была доступна из любой папки.

**Проверка:**

```bash
firebase --version
```

Должен вывести версию, например `13.x.x`. Если команда не находится, закройте и снова откройте терминал или проверьте, что Node.js установлен и путь к нему есть в переменной окружения PATH.

## 3. Вход в Firebase и выбор проекта

### 3.1. Вход в аккаунт Google

В терминале (из любой папки):

```bash
firebase login
```

- Откроется браузер.
- Войдите в тот же Google-аккаунт, в котором создан проект Firebase.
- После успешного входа в терминале появится сообщение вроде: «Success! Logged in as ...».

### 3.2. Привязка папки проекта к проекту Firebase

Перейдите в **корень вашего проекта** (там, где лежат папки `app`, `functions` и файл `firebase.json`). Например:

```bash
cd C:\StartDrive
```

Подставьте свой путь, если проект лежит в другой папке.

Затем укажите, с каким проектом Firebase работать:

```bash
firebase use startdrive-573fa
```

- `startdrive-573fa` — это **Project ID** вашего приложения.
- Узнать Project ID: Firebase Console → шестерёнка → **Настройки проекта** → в блоке «Ваши приложения» или в самом верху страницы будет **Project ID**.
- Если проект называется иначе — замените `startdrive-573fa` на ваш Project ID.

Успешный вывод: `Now using project startdrive-573fa`.

**Проверить текущий проект:**

```bash
firebase use
```

## 4. Установка зависимостей для функций

Код функции в папке `functions` использует библиотеки `firebase-admin` и `firebase-functions`. Их нужно установить через npm.

**Из корня проекта выполните:**

```bash
cd functions
npm install
cd ..
```

Пояснение:

- `cd functions` — переход в папку с кодом функции.
- `npm install` — установка зависимостей из `functions/package.json` (создаётся папка `node_modules`).
- `cd ..` — возврат в корень проекта (оттуда потом запускается деплой).

**Если появляются ошибки:**

- Убедитесь, что в папке `functions` есть файлы `package.json` и `index.js`.
- Попробуйте: `npm install --legacy-peer-deps`.
- Проверьте интернет-соединение.

После успешной установки в `functions` должна появиться папка `node_modules`.

## 5. Регион функции и Realtime Database

Функция срабатывает при записи в Realtime Database. Регион функции лучше совпадать с регионом базы (меньше задержка и не будет ошибок из-за несовпадения).

**Где посмотреть регион Realtime Database:**

1. Firebase Console → **Build** → **Realtime Database**.
2. Вверху страницы указан URL базы, например:  
   `https://startdrive-573fa-default-rtdb.europe-west1.firebasedatabase.app`  
   Регион здесь — **europe-west1** (часть перед `.firebasedatabase.app`).
3. Другой пример: если в URL есть `us-central1`, регион — **us-central1**.

**Где указан регион в коде:**

Откройте файл `functions/index.js`. Найдите блок:

```javascript
{
  ref: "/chats/{roomId}/messages/{messageId}",
  region: "europe-west1",
}
```

- Если ваша база в **europe-west1** — ничего менять не нужно.
- Если база в другом регионе (например **us-central1**) — замените на:

  `region: "us-central1"`

Сохраните файл.

## 6. Развёртывание функций

**Из корня проекта** (где лежит `firebase.json`) выполните:

```bash
firebase deploy --only functions
```

Пояснение:

- `firebase deploy` — загрузка и публикация кода в Firebase/Google Cloud.
- `--only functions` — разворачиваются только функции, без правил базы и хостинга.

**Что происходит при деплое:**

1. Локальная папка `functions` упаковывается и отправляется в облако.
2. В проекте создаётся (или обновляется) функция с именем `onNewChatMessage`.
3. Для неё настраивается триггер: создание данных по пути `/chats/{roomId}/messages/{messageId}` в Realtime Database.

**Успешный вывод выглядит примерно так:**

```
=== Deploying to 'startdrive-573fa' ...

i  deploying functions
i  functions: preparing codebase default for deployment
✔  functions: codebase default prepared for deployment
i  functions: ensuring required API cloudfunctions.googleapis.com is enabled...
i  functions: ensuring required API cloudbuild.googleapis.com is enabled...
✔  functions: required API cloudbuild.googleapis.com is enabled
i  functions: uploading codebase default...
i  functions: updating Node.js 20 function onNewChatMessage(europe-west1)...
✔  functions[onNewChatMessage]: Successful update operation.
✔  functions: all functions deployed successfully.
```

**Типичные ошибки и что делать:**

| Ошибка | Что проверить / сделать |
|--------|---------------------------|
| `HTTP Error 403` / `Permission denied` | Войти в Firebase: `firebase login`. Убедиться, что вы владелец/редактор проекта. |
| `Billing account not configured` | Включить тариф Blaze и привязать платёжный аккаунт (раздел 1.2). |
| `Project must be on Blaze plan` | То же — перейти на Blaze. |
| Ошибки про `region` / базу данных | Проверить, что в `index.js` указан тот же регион, что и у Realtime Database (раздел 5). |
| Ошибки при сборке (npm) | Выполнить в папке `functions`: `npm install` и снова `firebase deploy --only functions`. |

После успешного деплоя функция появится в консоли: **Build** → **Functions** — в списке должна быть **onNewChatMessage** (регион и триггер будут указаны там же).

## 7. Проверка работы

1. **Два пользователя:** на двух устройствах (или эмулятор + телефон) войдите в приложение под разными учётными записями.
2. **Чат:** откройте чат между этими пользователями и отправьте сообщение с одного аккаунта.
3. **Уведомление:** на устройстве **получателя** приложение должно быть в фоне или закрыто. Через несколько секунд должно прийти push-уведомление «Новое сообщение» с текстом сообщения и звуком.

**Если уведомление не приходит:**

- **Functions:** в Firebase Console → **Functions** → **onNewChatMessage** — нет ли ошибок в логах (вкладка «Журнал» / Logs).
- **Realtime Database:** в консоли откройте **Realtime Database** → **Data**. После отправки сообщения в чате под `chats` должна появиться запись с `roomId` и внутри — `messages` с новым сообщением. Если записи нет — сообщения не доходят до базы (проверьте правила и работу приложения).
- **Правила базы:** должны быть настроены как в `database.rules.json` (чтение/запись для авторизованных пользователей в `chats` и `presence`).
- **FCM-токен получателя:** при входе в приложение токен сохраняется в Firestore в документе пользователя (`users/{userId}`) в поле `fcmToken`. У получателя должен быть хотя бы один успешный вход после установки приложения и разрешение уведомлений.
- **Звук:** на устройстве не должна быть включена «тихая» или «не беспокоить» для приложения, канал уведомлений «Сообщения чата» (startdrive_chat) должен быть с звуком.

## 9. Что развёрнуто и где смотреть

- **Код функции:** локально в папке `functions/index.js`.
- **Развёрнутая функция:** в Google Cloud / Firebase — отображается в [Firebase Console](https://console.firebase.google.com) → ваш проект → **Build** → **Functions**.
- **Логи вызовов:** там же, у функции **onNewChatMessage** — вкладка с логами (ошибки, таймауты, успешные отправки FCM).

Если нужно изменить текст уведомления, звук или условия отправки — правьте `functions/index.js` и снова выполните `firebase deploy --only functions`.
