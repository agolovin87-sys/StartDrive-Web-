# Подробная пошаговая инструкция: устранение ошибок StartDrive

Выполняйте шаги по порядку. Если шаг уже выполнен (например, Firebase CLI установлен), переходите к следующему.

---

## Часть A. Индексы Firestore (ошибка «The query requires an index»)

### Шаг A1. Проверить и установить Node.js

1. Нажмите **Win + R**, введите `cmd`, нажмите Enter (откроется командная строка).
2. Введите команду и нажмите Enter:
   ```text
   node --version
   ```
3. Если появится номер версии (например, `v20.10.0`) — Node.js установлен, переходите к шагу A2.
4. Если команда не найдена:
   - Откройте в браузере: https://nodejs.org
   - Скачайте версию **LTS** (рекомендуемая).
   - Установите, оставив все галочки по умолчанию.
   - **Закройте и снова откройте** командную строку, затем снова выполните `node --version`.

### Шаг A2. Установить Firebase CLI

1. В командной строке выполните (можно копировать по одной строке):
   ```text
   npm install -g firebase-tools
   ```
2. Дождитесь окончания установки (в конце не должно быть красных строк с ошибками).
3. Проверьте установку:
   ```text
   firebase --version
   ```
   Должен появиться номер версии (например, `13.0.0`).

### Шаг A3. Войти в аккаунт Google через Firebase CLI

1. В той же командной строке выполните:
   ```text
   firebase login
   ```
2. Откроется браузер. Если появится запрос «Firebase wants to access your Google Account» — нажмите **Разрешить** (или **Allow**).
3. Войдите в **тот же** Google-аккаунт, под которым создан проект в Firebase Console.
4. В браузере должно появиться сообщение вроде «Success! You can close this window».
5. Вернитесь в командную строку — там должно быть написано, что вы вошли в систему.

### Шаг A4. Перейти в папку проекта

1. В командной строке введите (замените путь на свой, если проект лежит в другой папке):
   ```text
   cd C:\StartDrive
   ```
2. Нажмите Enter. Строка приглашения должна измениться и показывать путь к папке, например: `C:\StartDrive>`.

### Шаг A5. Привязать проект Firebase (если ещё не привязан)

1. Выполните:
   ```text
   firebase projects:list
   ```
2. В списке найдите проект **startdrive-573fa**. Запомните, что он есть.
3. Выполните:
   ```text
   firebase use startdrive-573fa
   ```
4. Должно появиться сообщение вроде «Now using project startdrive-573fa».
   - Если пишет «No project active» или просит выбрать проект — введите номер проекта из списка или снова выполните `firebase use startdrive-573fa`.

### Шаг A6. Задеплоировать индексы Firestore

1. Убедитесь, что вы по-прежнему в папке `C:\StartDrive` (команда `cd C:\StartDrive` из шага A4).
2. Выполните:
   ```text
   firebase deploy --only firestore:indexes
   ```
3. В выводе должно быть:
   - «Deploying to project 'startdrive-573fa'»
   - «firestore: released indexes...» или «firestore: deployed indexes...»
   - В конце: «Deploy complete!» или «✔ Deploy complete!»
4. Если появится ошибка «Permission denied» или «Not authorized» — убедитесь, что вошли под тем аккаунтом, у которого есть доступ к проекту в Firebase Console.

### Шаг A7. Проверить индексы в Firebase Console

1. Откройте браузер и перейдите по адресу: https://console.firebase.google.com
2. В списке проектов выберите **startdrive-573fa** (клик по названию).
3. В левом меню нажмите **Build** → **Firestore Database** (или «База данных Firestore»).
4. Вверху откройте вкладку **Indexes** (или «Индексы»).
5. В таблице должны быть индексы для коллекций **driving_sessions** и **instructor_open_windows**.
6. В колонке статуса может быть **Building** — подождите 2–5 минут и обновите страницу (F5). Статус должен смениться на **Enabled** (Включён).
7. Когда нужные индексы в статусе **Enabled**, закройте вкладку.

### Шаг A8. Перезапустить приложение

1. Полностью закройте приложение StartDrive на телефоне или эмуляторе (снять из списка недавних или остановить запуск в Android Studio).
2. Снова запустите приложение (**Run** в Android Studio или кнопка запуска).
3. Откройте кабинет инструктора → вкладку «Запись». Ошибки «The query requires an index» в логах больше не должно быть (после того как индексы стали Enabled).

**Вариант без Firebase CLI (только через браузер):**

1. Запустите приложение и откройте логи (Logcat в Android Studio).
2. Найдите в логе строку с текстом `create_composite` и ссылкой на `firestore/indexes` (длинный URL).
3. Скопируйте **целиком** первую такую ссылку и откройте её в браузере.
4. На странице нажмите кнопку **Create index** (Создать индекс).
5. Дождитесь создания индекса (несколько минут).
6. Повторите пункты 2–5 для **второй** ссылки в логе (для другой коллекции).
7. Перезапустите приложение.

## Часть B. Ошибка Google (DEVELOPER_ERROR / Unknown calling package)

### Шаг B1. Открыть папку проекта в командной строке

1. Нажмите **Win + R**, введите `cmd`, Enter.
2. Выполните (путь замените на свой, если проект в другой папке):
   ```text
   cd C:\StartDrive
   ```
3. Убедитесь, что в этой папке есть файл **gradlew.bat** (можно проверить командой `dir gradlew.bat`).

### Шаг B2. Получить отчёт о подписывании (signing report)

**Способ 1 — через командную строку:**

1. В папке `C:\StartDrive` выполните:
   ```text
   gradlew.bat signingReport
   ```
2. Подождите окончания сборки (первый раз может занять 1–2 минуты).
3. В выводе найдите блок, который начинается с **Variant: debug** (или **Variant: release**, если собираете release).
4. В этом блоке найдите строку **SHA1:** и скопируйте значение — это последовательность из букв и цифр в формате `XX:XX:XX:...` (например, `E1:2A:3B:4C:5D:...`). Скопируйте **всю** строку до конца, без пробелов в начале и конце.

**Способ 2 — через Android Studio:**

1. Откройте проект StartDrive в Android Studio.
2. Справа откройте панель **Gradle** (иконка слона или Gradle).
3. В дереве раскройте: **StartDrive** → **app** → **Tasks** → **android**.
4. Двойной клик по задаче **signingReport**.
5. Внизу откроется вкладка **Run** с выводом. Дождитесь окончания выполнения.
6. В выводе найдите блок **Variant: debug** и в нём строку **SHA1:**. Выделите значение SHA1 и скопируйте (Ctrl+C).

### Шаг B3. Открыть настройки проекта в Firebase Console

1. Откройте браузер и перейдите: https://console.firebase.google.com
2. В списке проектов нажмите на **startdrive-573fa**.
3. Рядом с названием проекта **«startdrive-573fa»** вверху страницы нажмите на иконку **шестерёнки** (⚙️) — «Project settings» / «Настройки проекта».

### Шаг B4. Найти Android-приложение и добавить отпечаток

1. Прокрутите страницу настроек вниз до раздела **Your apps** (Ваши приложения).
2. Если в списке уже есть приложение с платформой **Android** и пакетом **com.example.startdrive** — переходите к пункту 5.
3. Если Android-приложения нет: нажмите кнопку **Add app** (Добавить приложение), выберите иконку **Android**.
4. В форме укажите:
   - **Android package name:** `com.example.startdrive` (должно совпадать с пакетом в проекте).
   - **App nickname** — можно оставить пустым или вписать «StartDrive».
   - **Debug signing certificate SHA-1** — вставьте сюда скопированный SHA-1 из шага B2 и нажмите **Register app**. Дальше можно нажать «Next» и «Continue to console». После этого переходите к шагу B5.
5. Если приложение уже есть: в карточке этого приложения найдите надпись **SHA certificate fingerprints** и нажмите **Add fingerprint** (Добавить отпечаток).
6. В появившемся поле вставьте скопированный SHA-1 (Ctrl+V) и нажмите **Save** (Сохранить).
7. Убедитесь, что отпечаток появился в списке под приложением.

### Шаг B5. Скачать новый файл google-services.json

1. Оставаясь на странице настроек проекта (Project settings), в карточке Android-приложения найдите кнопку **Download google-services.json** (или иконку скачивания рядом с названием приложения).
2. Нажмите её — файл **google-services.json** скачается в папку «Загрузки» (или ту, что указана в браузере).
3. Откройте проводник Windows и перейдите в папку проекта: **C:\StartDrive\app**.
4. Найдите в папке **app** существующий файл **google-services.json**.
5. Переместите или скопируйте только что скачанный **google-services.json** из папки «Загрузки» в **C:\StartDrive\app**, выбрав **Заменить** существующий файл.
6. Убедитесь, что в `C:\StartDrive\app` лежит один файл **google-services.json** и он новый (по дате изменения).

### Шаг B6. Очистить и пересобрать проект в Android Studio

1. Откройте проект StartDrive в Android Studio.
2. В меню выберите **Build** → **Clean Project** и дождитесь окончания.
3. Затем **Build** → **Rebuild Project** и дождитесь окончания (без ошибок).
4. В меню **Run** → **Run 'app'** (или зелёная кнопка Run / Shift+F10), чтобы запустить приложение на подключённом устройстве или эмуляторе.

### Шаг B7. Проверить результат

1. После запуска приложения откройте логи (вкладка **Logcat** внизу в Android Studio).
2. В фильтре можно ввести `startdrive` или оставить без фильтра.
3. Ошибок **DEVELOPER_ERROR**, **Unknown calling package name** и **Failed to get service from broker** в логах быть не должно (или они появятся реже и не блокируют работу).

**Если ошибка осталась:** убедитесь, что вы добавили SHA-1 именно того варианта сборки, который запускаете (debug или release), и что заменили **app/google-services.json** новым файлом после добавления отпечатка. Для release-сборки нужно добавить SHA-1 release-ключа и при необходимости обновить google-services.json ещё раз.

## Часть C. Остальные сообщения в логе

- **«Target id not found: 14»** (или другой номер) в строке с **WatchStream** — это внутреннее сообщение Firestore. Обычно исчезает после выполнения части A (индексы) и перезапуска приложения. Отдельно ничего делать не нужно.
