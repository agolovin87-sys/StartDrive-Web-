# Если при загрузке аватара: «Нет соединения с интернетом»

## 1. Откуда открыто приложение

- **Не открывайте** сайт по адресу `file:///...` (файл с диска).
- Открывайте по адресу **http://localhost:ПОРТ** или ваш реальный домен (например, после `npm run serve` или развёртывания).

## 2. Жёсткое обновление и кэш

- Нажмите **Ctrl+Shift+R** (или Cmd+Shift+R на Mac), чтобы обновить страницу без кэша.
- Либо откройте сайт в режиме **инкогнито** и попробуйте загрузить аватар снова.

## 3. Что видно в консоли браузера

1. Нажмите **F12** → вкладка **Console**.
2. Снова попробуйте загрузить аватар (Выбрать фото → Готово).
3. Найдите в консоли строку **«StartDrive avatar upload error:»** и посмотрите текст ошибки после неё.

- **Failed to fetch** / **NetworkError** — чаще всего CORS или сеть. Ниже — что проверить по CORS.
- **403** / **permission-denied** — проблема с правилами Storage или авторизацией.
- Другой текст — скопируйте его для диагностики.

## 4. CORS уже настроен?

Убедитесь, что скрипт CORS для Storage уже выполнялся и прошёл без ошибок:

```powershell
cd c:\StartDrive\functions
node scripts/set-storage-cors.js
```

Должно быть сообщение: **«CORS установлен для бакета: startdrive-573fa.firebasestorage.app»**.

## 5. С какого адреса идёт запрос

В **F12** → вкладка **Network** (Сеть):

1. Включите запись (кнопка записи должна быть активна).
2. Снова загрузите аватар.
3. Найдите запрос с ошибкой (красный или статус 0/CORS).
4. Посмотрите **URL** этого запроса (например, `firebasestorage.googleapis.com` или `upload.googleapis.com`) и **статус** (403, 0, CORS blocked и т.п.).

Если в консоли или в Network видно **CORS** или **blocked by CORS** — ещё раз выполните скрипт CORS (п. 4) и сделайте жёсткое обновление страницы (п. 2).

## 6. Расширения браузера

Отключите на время проверки блокировщики рекламы и другие расширения, которые могут резать запросы к Google/Firebase.
