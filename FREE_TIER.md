# Использование без платного тарифа (Spark)

Можно пользоваться приложением и чатом **на бесплатном тарифе Firebase (Spark)** — платить не нужно.

## Что работает бесплатно

- **Вход и регистрация** (Firebase Auth).
- **Данные пользователей, баланс, роли** (Firestore).
- **Чат в реальном времени** (Realtime Database):
  - сообщения сохраняются и синхронизируются между пользователями;
  - при открытой вкладке «Чат» новые сообщения приходят сразу;
  - **звук при новом сообщении** от собеседника воспроизводится в приложении.
- **Правила безопасности** Realtime Database можно публиковать на Spark — настройте их в консоли по `database.rules.json` / `database.rules.console.txt`.

## Чего не будет без платного тарифа

- **Push-уведомления при закрытом или свёрнутом приложении** (в шторку, на экран блокировки).  
  Для этого нужна Cloud Function, которая срабатывает при новом сообщении и отправляет FCM. Cloud Functions в Firebase доступны только на тарифе **Blaze**. Без Blaze такие уведомления настроить через Firebase нельзя.

## Что сделать, чтобы чат работал без оплаты

1. Оставьте проект на тарифе **Spark**.
2. В [Firebase Console](https://console.firebase.google.com) включите **Realtime Database** (если ещё не включена) и в разделе **Rules** вставьте правила из файла `database.rules.json` (или из `database.rules.console.txt`), затем нажмите **Publish**.
3. Cloud Functions разворачивать не обязательно: команда `firebase deploy --only functions` нужна только для push-уведомлений при закрытом приложении. На тарифе Spark эта команда всё равно не сработает (функции доступны только на Blaze).
4. Соберите и установите приложение — чат и звук при новом сообщении (пока приложение открыто) будут работать.

Итог: **без платного тарифа сообщения доходят в реальном времени и со звуком, пока пользователь в приложении.** Уведомления на заблокированный экран требуют тарифа Blaze и развёртывания функций (см. `DEPLOY_FUNCTIONS.md`).
