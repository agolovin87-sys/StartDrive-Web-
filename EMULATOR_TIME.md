# Время в эмуляторе не совпадает с реальным — что делать

Если время в симуляторе Android «уехало», попробуйте варианты по порядку.

---

## Вариант 1: Установить время через ADB (эмулятор с root)

Эмулятор обычно запущен с root, поэтому эта команда часто срабатывает.

**Формат времени:** `MMDDhhmmГГГГ.сс` (месяц, день, час, минута, год, секунды).

Откройте терминал (PowerShell или cmd) и выполните **одну** из команд.

Текущее время с вашего ПК подставится само (PowerShell):

```powershell
adb shell su root date $(Get-Date -UFormat "%m%d%H%M%Y.%S")
```

Если не сработало — подставьте время вручную. Пример: **28 февраля 2025, 15:30:45**:

```bash
adb shell su root date 022815302025.45
```

Проверка времени в эмуляторе:

```bash
adb shell date
```

---

## Вариант 2: Только дата (без root)

На некоторых образах срабатывает установка без `su`:

```bash
adb shell date -s 202502281530.45
```

Формат: `ГГГГММДДччмм.сс`. Если команда выдаёт ошибку — используйте вариант 1 или 3.

---

## Вариант 3: Настройки эмулятора + принудительная синхронизация

1. В эмуляторе: **Settings (Настройки)** → **System** → **Date & time**.
2. Выключите **Automatic date & time**.
3. Поставьте **правильную дату и время** вручную.
4. Включите **Automatic time zone** и выберите ваш часовой пояс.
5. Снова включите **Automatic date & time** — устройство подтянет время по сети (NTP).
6. Убедитесь, что у эмулятора есть доступ в интернет (в настройках эмулятора или через Wi‑Fi в образе).

---

## Вариант 4: Сменить NTP-сервер (если есть root)

По умолчанию Android использует `time.android.com`. Можно явно задать сервер:

```bash
adb shell settings put global ntp_server time.google.com
```

После этого в настройках включите **Automatic date & time** и подождите минуту.

---

## Вариант 5: Холодная перезагрузка эмулятора

Иногда время «залипает» из-за снимка (snapshot):

1. Закройте эмулятор полностью.
2. В Android Studio: **Tools** → **Device Manager**.
3. У нужного устройства нажмите **⋮** → **Cold Boot Now** (загрузка без восстановления снапшота).
4. После загрузки проверьте время; при необходимости снова включите **Automatic date & time** в настройках.

---

## Вариант 6: Отключить Quick Boot (сохранять время между запусками)

Если используете Quick Boot, эмулятор может восстанавливать старое время из снапшота:

1. **Device Manager** → **⋮** у устройства → **Edit**.
2. В разделе **Emulated Performance** найдите **Quick Boot**.
3. Выберите **Cold boot** вместо **Quick boot** и сохраните.
4. Запустите эмулятор заново и при необходимости установите время по варианту 1 или 3.

---

## Вариант 7: Тестировать на реальном устройстве

Если для вас важно точное время (чаты, история, уведомления):

1. Подключите телефон по USB.
2. Включите **Режим разработчика** и **Отладку по USB**.
3. В Android Studio выберите этот телефон вместо эмулятора и запустите приложение.

Время на телефоне обычно синхронизируется с сетью автоматически.

---

## Краткая шпаргалка по форматам

| Способ              | Формат примера        | Пример (28.02.2025 15:30:45)   |
|---------------------|------------------------|---------------------------------|
| `su root date`      | MMDDhhmmГГГГ.сс       | `022815302025.45`              |
| `date -s` (если есть) | ГГГГММДДччмм.сс     | `202502281530.45`              |

Проверить, какая утилита есть: `adb shell which date` и `adb shell date`.
